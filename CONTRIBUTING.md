# Contributing to Build a Brain

Thanks for wanting to help raise the mind! 🧠 This project is a teaching tool, so the bar is a
little different from a normal codebase: **clarity and correctness beat cleverness.**

## The golden rules

1. **The math must be correct.** Verify every number by hand *and*, where possible, against a
   library (numpy/PyTorch). Wrong-but-confident is the worst outcome for a learning resource.
2. **Stay in the story voice.** The whole curriculum is one narrative — *raising a mind*. Read
   [`STORY_GUIDE.md`](STORY_GUIDE.md) before writing content. Open on a real-world scene, explain
   *why* before *what*, and show formulas three ways: plain words → worked example by hand → verified.
3. **Keep numbers tiny.** Worked examples use 2–3 element vectors so a reader can follow every
   multiply and add.

## Working on the web app

```bash
cd webapp
npm install
npm run dev        # http://localhost:6023
npm run build      # ALWAYS run this before opening a PR — it type-checks the whole app
```

- 2-D interactives live in `src/lessons/`; lazy-loaded 3-D scenes in `src/three/`.
- Chapter prose/interactives are in `src/pages/` (`articles.tsx` + `lessonContent.tsx`); curriculum
  metadata is in `src/lessons.ts`.
- **r3f gotcha:** never call `setState` during render or `useMemo` in a Three.js component — use
  `useEffect`.

## Working on the notebooks

Notebooks in `playbook/` are generated by builder scripts in `_builders/` — **edit the builder, not
the `.ipynb` directly**, then rebuild and execute so verified outputs are embedded:

```bash
python _builders/playbook/build_P00.py
python -m jupyter nbconvert --to notebook --execute --inplace \
       --ExecutePreprocessor.kernel_name=tinygpt playbook/P00_*.ipynb
```

When editing notebook prose, change only the markdown cells; leave code cells (and their by-hand
numbers) intact unless you're deliberately fixing the math.

## Pull requests

- One topic per PR; describe what you changed and why.
- Confirm `npm run build` passes (webapp) and any touched notebook executes cleanly.
- Be kind in review — this is a place for people learning out loud.
