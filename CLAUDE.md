# CLAUDE.md — tiny_llm project

This repo has **three** things. Detailed, up-to-date notes live in the auto-memory
(`.claude/projects/.../memory/` — `project-overview.md`, `environment-setup.md`, `playbook.md`,
`webapp.md`); read those for specifics. Quick map:

## 1. TinyGPT notebooks — `notebooks/`
A from-scratch GPT built step by step (`00`–`13`), plan in `TINYGPT_LEARNING_ROADMAP.md`.
Teaching style: every formula shown 3 ways — plain words → tiny worked example **by hand** → code
that re-computes & verifies the numbers. Keep numbers tiny.

## 2. The Playbook — `playbook/` (P00–P29) + `PLAYBOOK_ROADMAP.md`
A complete from-zero ML/DL/NLP/LLM curriculum that leads into the TinyGPT notebooks. Each notebook is
generated by a builder script then executed to embed verified outputs. Same by-hand-then-verified
style, with an interactive "📊 Step 4 — See it & play with it" cell (matplotlib + ipywidgets) on the
math-heavy ones.

- Builders: `_builders/playbook/build_PNN.py` + shared `_pbcommon.py` (`md/code/readonly/build`).
- Build + execute a notebook (note: **runs on Anaconda python, kernel `tinygpt`** — NOT `.venv`):
  ```bash
  PY="$LOCALAPPDATA/anaconda3/python.exe"
  "$PY" _builders/playbook/build_P00.py
  "$PY" -m jupyter nbconvert --to notebook --execute --inplace \
        --ExecutePreprocessor.kernel_name=tinygpt playbook/P00_*.ipynb
  ```
- Installed: numpy, pandas, sklearn, torch (CPU), matplotlib, seaborn, scipy, ipywidgets.
  **Blocked by enterprise security:** tensorflow, keras, transformers, peft, trl → shown as
  READ-ONLY (`skip-execution` tagged) cells. **sklearn `KMeans` crashes** (threadpoolctl/OpenBLAS) —
  use from-scratch numpy instead.

## 3. The Web App — `webapp/` ("Build a Brain")
Interactive, story-driven site (React 19 + Vite + TypeScript + Three.js), the playbook as one
narrative "from a single number to a thinking machine". Inspired by MLU-Explain, TF Playground,
3Blue1Brown. **npm WORKS here** (unlike pip).

```bash
cd webapp
npm install          # one-time
npm run dev          # http://localhost:6023  (fixed port, strictPort; preview uses 6023 too)
npm run build        # type-check + bundle -> dist/ (host-agnostic: base "./" + hash router)
npm run preview      # serve the built dist
```

Architecture: `src/lessons.ts` (curriculum + story + `built` flag) · `src/lessons/*` (2D interactives)
· `src/three/*` (lazy 3D scenes; Three.js only ships to 3D chapters) · `src/pages/lessonContent.tsx`
(interactive lessons) + `src/pages/articles.tsx` (prose) merged into `CONTENT` · `src/store.ts`
(zustand: `usePersistentState` keeps figure settings across navigation) · `src/components/Controls.tsx`
(`Transport` Play/Pause/Step/Reset, `ViewSwitcher` 2D/3D tabs). Always `npm run build` to verify edits.
**r3f gotcha:** never call setState during render/useMemo — use useEffect.
